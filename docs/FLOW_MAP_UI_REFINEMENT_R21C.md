# R21C 项目实时流程地图 UI 布局重构

## 目标

按 `map2.md` 提示词重新整理项目实时流程地图的视觉层级和画布排版，让用户进入 `/projects/flow-map` 或 `/projects/:projectId/flow-map` 后能直接看到清晰、完整、可读的 18 步实时流程地图。

本轮只调整前端布局、视觉样式和浏览器验收，不改变已冻结业务状态机、流程规则、权限规则或后端接口语义。

## 页面范围

- `/projects/flow-map`
- `/projects/:projectId/flow-map`
- 共用组件：`FlowMapWorkspace`

## 布局调整

- 移除左侧常驻“流程地图控制台”，避免压缩主画布。
- 新增顶部工具栏，集中放置视图筛选、缩放、适应屏幕、重置位置和“图例 / 筛选”弹层。
- 画布固定为业务拓扑尺寸 `1440 x 1740`，默认按容器宽度自适应缩放，普通笔记本和大屏都能完整阅读。
- 主线固定居中，关键节点按业务流程自上而下排列。
- 第 12 步“样车驾驶室评审”改为紧凑菱形，文字不旋转，不遮挡第 14 步。
- 所有连线改为水平 / 垂直正交折线，去掉长斜线和跨屏曲线。
- 节点内容收敛为步骤号、节点名称、状态、责任部门 / 负责人、截止或专项摘要，避免卡片内部拥挤。

## 坐标规则

画布中心线为 `x=720`：

- 主线：第 1、2、3、4、6、10、11、12、14、15、16、17、18 步。
- 右侧并行：第 5、7、8 步。
- 左侧并行：第 9、13 步。
- 第 12 步退回第 11 步使用右侧正交回路线。
- 第 12 步通过后分别进入第 13 步和第 14 步，保持第 13 步非阻塞语义。

## 状态颜色规则

沿用 R21 已建立的统一状态色：

| 状态 | 中文文案 | 视觉规则 |
|---|---|---|
| `COMPLETED` | 已完成 | 绿色边框、浅绿底、绿色实线 |
| `COMPLETED_LATE` | 逾期完成 | 黄绿色强调 |
| `IN_PROGRESS` / `PENDING` | 进行中 / 待处理 | 蓝色边框、浅蓝底、当前路径高亮 |
| `PENDING_REVIEW` | 待评审 | 橙色边框、浅橙底 |
| `OVERDUE` | 已逾期 | 红色边框、红色提示和逾期天数 |
| `RETURNED` | 已退回 | 紫色节点和回路线 |
| `MONTHLY_TRACKING` | 月度跟踪 | 青色节点，显示月度完成进度 |
| `EXIT_PENDING` | 待退出 | 深橙强调 |
| `NOT_STARTED` | 未开始 | 灰色节点与灰色虚线 |

## 交互策略

- 默认显示全部节点。
- 筛选项仍支持：全部节点、只看主线、只看风险节点、只看我的任务、只看未完成。
- 筛选只改变地图高亮和淡化，不改变业务状态。
- 节点点击仍打开工序详情抽屉，URL `taskId` 可恢复。
- 顶部保留项目状态栏，展示当前节点、进度、负责人、责任部门、逾期节点、月度评审和最近更新。

## 自动刷新策略

- 沿用 R21：流程地图每 30 秒轮询刷新。
- 工序详情抽屉每 15 秒轮询刷新。
- 页面顶部显示“最近更新：YYYY-MM-DD HH:mm:ss”。
- 手动“立即刷新”继续可用。
- 用户执行工序动作时保留既有保护，不在提交过程中覆盖正在处理的数据。

## 验收覆盖

新增 Playwright 用例：

- `apps/web/e2e/r21c-flow-map-ui-refinement.spec.ts`

覆盖点：

- `/projects/flow-map` 全局入口可选择项目并显示 18 个节点。
- `/projects/:projectId/flow-map` 显示顶部工具栏和完整地图。
- 左侧常驻控制台不再渲染。
- “图例 / 筛选”弹层可打开。
- 第 12、17、18 步专项节点可见。
- 第 12 步与第 14 步不重叠。
- 全部节点不互相重叠。
- 所有 SVG 连线为正交折线。
- 1440 / 1920 / 移动端截图可生成。
- 节点详情抽屉可打开，刷新后 `taskId` 状态可恢复。
