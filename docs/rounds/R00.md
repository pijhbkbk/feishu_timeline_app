# R00：仓库审计 + 执行底座搭建

## 轮次目标
建立 Codex 自主执行底座，审计现有仓库结构、技术栈、可复用代码、缺失模块和命令基线，为后续自动推进提供稳定起点。

---

## 本轮输入
必须阅读：

- `AGENTS.md`
- `docs/EXECUTION_LEDGER.md`
- 三份正式业务文档
- 根目录 `README*`
- 根目录及关键子目录 `package.json`、`pnpm-workspace.yaml`、`turbo.json`、`nx.json`

---

## 本轮范围
允许修改：

- `AGENTS.md`
- `docs/EXECUTION_LEDGER.md`
- `docs/rounds/R00.md` ~ `R10.md`
- `docs/repo-audit.md`
- `docs/ARCHITECTURE_DECISIONS.md`
- `docs/ACCEPTANCE_CHECKLIST.md`
- `scripts/codex/*`
- 辅助性 README/脚本（仅限执行底座相关）

本轮不做：

- 业务功能开发
- 数据库正式建模
- 前后端正式业务代码实现

---

## 必做事项

1. 审计仓库当前结构与已有工程基础。
2. 识别：
   - monorepo / 单仓结构
   - 前端目录
   - 后端目录
   - 数据库目录
   - 测试目录
   - CI/CD 配置
   - Docker / 部署脚本
3. 输出 `docs/repo-audit.md`，至少说明：
   - 当前仓库概览
   - 已具备能力
   - 缺失能力
   - 与目标架构的差异
   - 后续执行建议
4. 输出或完善：
   - `docs/ARCHITECTURE_DECISIONS.md`
   - `docs/ACCEPTANCE_CHECKLIST.md`
   - `scripts/codex/run-round.sh`
   - `scripts/codex/continue.sh`
   - `scripts/codex/final-gate.sh`
5. 在 `docs/EXECUTION_LEDGER.md` 中写入 R00 执行结果。

## 停止条件
出现以下任一情况，停止并输出 blocker report：

- 仓库关键结构无法识别
- 安装依赖即发生严重冲突且无法在 2 次内修复
- 当前仓库与三份正式文档目标严重不兼容且需要重大重构决策

若本轮全部验收项通过，则：

- 在 `docs/EXECUTION_LEDGER.md` 中将 R00 标记为 `PASSED`
- 自动进入 `R01`

否则：

- 标记为 `FAILED` 或 `BLOCKED`
- 输出 blocker report
