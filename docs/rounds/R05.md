# R05：API 层与 OpenAPI 文档

## 轮次目标
将后端能力暴露为稳定 API，并生成可供前端和测试使用的 OpenAPI 文档。

---

## 本轮输入
必须阅读：

- `AGENTS.md`
- `docs/EXECUTION_LEDGER.md`
- `docs/rounds/R05.md`
- R03 / R04 服务与模块
- 业务文档中的页面与交互需求

---

## 本轮范围
允许修改：

- 控制器 / 路由层
- DTO / 校验器
- OpenAPI / Swagger 配置
- API 测试
- 权限接入
- 报表聚合接口

本轮不做：

- 前端正式页面实现
- 大规模前端可视化开发

---

## 必做事项

1. 建立项目接口：
   - 创建
   - 列表
   - 详情
   - 阶段概览
2. 建立节点接口：
   - 详情
   - 表单保存
   - 完成
   - 通过
   - 不通过
   - 退回
   - 历史轮次查询
3. 建立第 17 步月度评审接口。
4. 建立第 18 步颜色退出录入接口。
5. 建立基础报表接口。
6. 输出 OpenAPI / Swagger 文档。

---

## 期望产物

- 稳定 API
- DTO 与参数校验
- 权限接入
- OpenAPI 文档
- API 级测试
- 执行账本更新

---

## 验收标准

- [ ] OpenAPI 文档可打开
- [ ] 主业务接口可调通
- [ ] 权限错误码清晰
- [ ] 参数校验可用
- [ ] 幂等接口边界清晰
- [ ] 账本已更新

---

## 建议执行命令
```bash
pnpm --filter api build
pnpm --filter api test
pnpm --filter api start:dev || true
curl -fsS http://localhost:3001/api/docs || true
```

---

## 停止条件

- API 边界与后端领域模型发生根本冲突
- 权限中间件导致接口无法稳定工作
- OpenAPI 生成机制与现有架构冲突严重

---

## 本轮完成后的决策规则
若本轮全部验收项通过，则进入 `R06`。
