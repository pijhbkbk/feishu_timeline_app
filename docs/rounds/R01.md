# R01：工程底座与本地开发环境打通

## 轮次目标
打通本地开发环境、数据库容器、缓存容器、环境变量模板、统一开发命令与 health check，为后续数据库和业务开发提供稳定底座。

---

## 本轮输入
必须阅读：

- `AGENTS.md`
- `docs/EXECUTION_LEDGER.md`
- `docs/rounds/R01.md`
- `docs/repo-audit.md`
- 仓库已有启动脚本、环境变量模板、Docker 配置

---

## 本轮范围
允许修改：

- `docker-compose.yml` / `compose.dev.yml`
- `.env.example`
- `apps/api/.env.example`
- `apps/web/.env.example`
- `README*`
- `scripts/dev/*`
- `package.json` / 子包 `package.json`
- `scripts/codex/*`

本轮不做：

- 正式数据库建模
- 正式业务规则开发
- 正式前端业务页面开发

---

## 必做事项

1. 打通 PostgreSQL 与 Redis 的本地启动。
2. 建立统一本地开发命令，例如：
   - 安装依赖
   - 启动容器
   - 启动前端
   - 启动后端
   - 一键开发启动
3. 建立环境变量模板。
4. 若无 health check，则补齐基础健康检查接口。
5. 确认以下命令是否存在并可运行：
   - lint
   - typecheck
   - build
   - test
6. 将本轮实际使用的命令写入 `docs/EXECUTION_LEDGER.md`。

---

## 期望产物

- 本地开发容器配置
- 环境变量模板
- 开发启动命令文档

## 验收标准

- [ ] 本地 PostgreSQL 可启动
- [ ] 本地 Redis 可启动
- [ ] 前后端可分别启动
- [ ] 至少存在一个 health check
- [ ] 存在统一开发命令
- [ ] 环境变量模板完整
- [ ] 账本已更新

---

## 建议执行命令
```bash
docker compose up -d postgres redis || docker-compose up -d postgres redis
pnpm -r lint || true
pnpm -r typecheck || true
pnpm -r test || true
pnpm -r build || true
pnpm dev || true
```

---

## 停止条件

- 本地数据库无法启动且 2 次修复失败
- 环境变量缺失严重到无法推进后续开发
- 当前工程底座与目标栈严重冲突，需要架构级决策

---

## 本轮完成后的决策规则
若本轮全部验收项通过，则进入 `R02`。
