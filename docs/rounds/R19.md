# R19_SECURITY_AUDIT_FOR_PRIVATE_CLOUD_AND_FEISHU

## Goal

在正式部署到公司私有云、并上架飞书工作台之前，建立完整安全检查范围、检查清单、威胁模型和自动化安全脚本基线。当前子阶段只做范围确认与脚本准备，不执行全量扫描。

## Scope

- 新增 `docs/security/SECURITY_SCOPE_R19.md`
- 新增 `docs/security/SECURITY_CHECKLIST_R19.md`
- 新增 `docs/security/THREAT_MODEL_R19.md`
- 新增 `scripts/security/*`
- 更新 `docs/EXECUTION_LEDGER.md`
- 不扫描飞书开放平台自身域名。
- 不扫描公司未授权 IP。
- 不对生产做主动扫描。
- 不输出真实密钥、token、cookie、数据库密码、飞书 App Secret。

## Acceptance

- [ ] 安全范围文档覆盖业务模块、接口、数据、环境和禁止范围。
- [ ] 安全检查清单覆盖主机、SAST、SCA、密钥、DAST、认证、权限、输入输出、文件上传、网页防篡改、业务逻辑和飞书工作台。
- [ ] 威胁模型覆盖资产、攻击者、关键威胁、风险分级和安全目标。
- [ ] 基础脚本存在并可通过 shell 语法检查。
- [ ] 脚本默认优先本地或显式授权目标，不自动扫描生产。
- [ ] 台账记录本轮范围、改动、命令、风险和下一步决策。

## Next Gate

用户或公司信息安全负责人确认 `SECURITY_SCOPE_R19.md` 与 `SECURITY_CHECKLIST_R19.md` 后，才进入全量扫描和整改复测阶段。
