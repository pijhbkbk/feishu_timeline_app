# R02：数据库 Schema、迁移脚本、种子数据

## 轮次目标
将三份正式文档中的核心数据模型落地为数据库 Schema、迁移脚本和种子数据，并保证空库可重复初始化。

---

## 本轮输入
必须阅读：

- `AGENTS.md`
- `docs/EXECUTION_LEDGER.md`
- `docs/rounds/R02.md`
- 《字段字典 + 数据库表设计》
- 《流程规则配置表 + 状态机说明》
- 现有 ORM / migration / seed 代码

---

## 本轮范围
允许修改：

- ORM schema 文件（如 Prisma schema）
- migration 目录
- seed 脚本
- 枚举定义
- 索引/约束配置
- 初始化数据脚本
- 数据库 README / 文档

本轮不做：

- 流程引擎正式服务实现
- 前端业务页面实现

---

## 必做事项

1. 建立或完善以下核心表：
   - `users`
   - `roles`
   - `departments`
   - `role_permissions`
   - `projects`
   - `process_templates`
   - `node_definitions`
   - `process_instances`
   - `node_instances`
   - `node_form_data`
   - `attachments`
   - `approval_records`
   - `notification_logs`
   - `audit_logs`
   - `work_calendar`
   - `recurring_plans`
   - `recurring_tasks`
   - `system_parameters`
2. 初始化 18 个节点定义。
3. 初始化关键参数：
   - 颜色开发收费固定金额 10000
   - 第 17 步月度周期 12 次
   - 年产量退出阈值
4. 初始化基础角色、部门、权限。
5. 建立工作日历导入或初始化机制。
6. 保证空库可一键 migrate + seed。

---

## 期望产物

- 完整 schema
- migration 脚本
- seed 脚本
- 节点定义初始化数据
- 参数初始化数据
- 账本更新

---

## 验收标准

- [ ] 空库可成功迁移
- [ ] 空库可成功 seed
- [ ] `node_definitions` 中有 18 个节点
- [ ] 第 4、6、12、16 节点的规则初始化到位
- [ ] 固定收费金额 10000 已初始化
- [ ] 数据库约束可防止明显脏数据
- [ ] 账本已更新

---

## 建议执行命令
```bash
pnpm --filter api exec prisma validate
pnpm --filter api exec prisma format
pnpm --filter api exec prisma migrate dev --name init_color_pm_core
pnpm --filter api exec prisma db seed
pnpm --filter api exec prisma studio || true
```

---

## 停止条件

- 迁移涉及破坏性删除且无法安全处理
- 现有 schema 与目标模型冲突严重且需重构决策
- seed 连续 2 次失败且原因不明

---

## 本轮完成后的决策规则
若本轮全部验收项通过，则进入 `R03`。
