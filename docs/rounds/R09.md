# R09：部署脚本、CI/CD、监控、备份、预发布

## 轮次目标
完成容器化、CI/CD、staging 部署、备份、回滚、健康检查与监控接入，使系统进入“可部署、可回滚、可巡检”的预发布状态。

---

## 本轮输入
必须阅读：

- `AGENTS.md`
- `docs/EXECUTION_LEDGER.md`
- `docs/rounds/R09.md`
- 当前 Docker / deploy / CI 配置
- 现有 GCE 或目标部署环境约束

---

## 本轮范围
允许修改：

- Dockerfile
- docker-compose / deploy compose
- CI 工作流
- deploy 脚本
- migrate / seed / rollback 脚本
- 健康检查脚本
- 监控与日志接入配置
- staging 部署文档

本轮不做：

- 正式 UAT 结论收口
- 正式上线版本发布

---

## 必做事项

1. 前后端容器化。
2. 建立 staging 一键部署脚本。
3. 分离 migrate 与 seed 脚本职责。
4. 建立 release / rollback 脚本。
5. 建立健康检查与基本监控。
6. 输出 staging 部署文档。

---

## 期望产物

- Dockerfile
- deploy 配置
- CI/CD 配置
- health-check 脚本
- rollback 脚本
- staging 文档
- 执行账本更新

---

## 验收标准

- [ ] staging 可从空环境启动
- [ ] build / migrate / start 流程可执行
- [ ] 日志可查
- [ ] rollback 脚本存在
- [ ] README / deploy 文档完善
- [ ] 账本已更新

---

## 建议执行命令
```bash
docker build -t color-pm-web -f apps/web/Dockerfile .
docker build -t color-pm-api -f apps/api/Dockerfile .
docker compose -f deploy/compose.staging.yml up -d || true
bash scripts/deploy/health-check.sh || true
bash scripts/deploy/rollback-check.sh || true
```

---

## 停止条件

- 部署环境关键变量缺失或权限不足
- 迁移/回滚脚本存在明显数据风险
- 容器化策略与现有仓库结构严重冲突

---

## 本轮完成后的决策规则
若本轮全部验收项通过，则进入 `R10`。
