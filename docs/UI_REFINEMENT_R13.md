# UI_REFINEMENT_R13.md

## 目标

R13 只做系统化精修，不推翻既有页面结构，不改动冻结业务规则。收口重点是：

- 统一页面标题、按钮、状态色和反馈组件
- 提升第 12、17、18 步的阅读效率与关键动作可理解性
- 明确 recurring task 在流程截止日历中的展示边界
- 为 Playwright 浏览器级回归提供稳定定位点和可重复执行脚本

## 标题层级

- 顶栏页面标题：`.topbar-title`
  用于一级页面语义，如“流程”“评审”“颜色退出”。
- 页面卡片标题：`.section-title`
  用于工作区级模块标题，如“驾驶室评审表单”“第 17 步月度评审台账”。
- 明细块标题：`detail-block h3`
  用于摘要、说明、绑定规则、退出建议等二级说明块。
- 辅助眉标：`.eyebrow`
  仅用于路由语义和英文短标签，不承载主要业务信息。

## 按钮层级

- 主按钮：`.button.button-primary`
  用于保存、通过、创建等正向主动作。
- 次按钮：`.button.button-secondary`
  用于刷新、跳转、编辑、提交、查看详情等次级动作。
- 危险按钮：`.button.button-danger`
  用于驳回、完成颜色退出等带明显不可逆或回退语义的动作。
- 小按钮：`.button.button-small`
  仅用于表格动作列，不用于主表单提交。

## 状态颜色体系

- 未开始：灰蓝 `#94a3b8`
- 进行中：蓝色 `#2563eb`
- 待评审：橙色 `#c2410c`
- 已完成：绿色 `#15803d`
- 已逾期：红色 `#b91c1c`
- 已退回：紫色 `#7c3aed`

对应组件：

- `status-pill-*`
- `workflow-node-*`
- `monthly-review-card-*`
- `overdue-badge`

## 间距与容器统一

- 页面卡片统一使用 `30px` 内边距，移动端降到 `24px`
- 卡片、表格、时间线、详情块统一圆角 `20px`
- 表单间距统一使用 `16px`
- 摘要卡、元数据卡统一使用 `summary-card` / `metadata-item`
- 长表格统一使用 `table-shell table-shell-scroll`
  表头固定，正文滚动，避免月度台账和评审历史挤压页面

## 空状态、错误状态、无权限状态

- 统一使用 `StatePanel`
- 变体：
  - `empty`
  - `error`
  - `permission`
  - `loading`
- 登录缺失提示、月度台账空状态、评审详情未选中、退出摘要未生成都走同一视觉结构

## Toast / Loading / Disabled 规范

- 页内成功、失败、风险提示统一使用 `FeedbackBanner`
- `error` 使用红底浅色块，`success` 使用绿底浅色块，`warning` 用橙底浅色块
- loading 仍沿用按钮文案切换，如“刷新中…”“处理中…”
- disabled 按钮统一降透明度，不再仅依赖浏览器默认灰态

## 关键页面精修结果

### 第 12 步评审工作区

- 通过、提交、驳回动作层级已经区分
  - 通过：主按钮
  - 提交：次按钮
  - 驳回：危险按钮
- 评审表单上方新增结论提示卡，直接提示“条件通过/驳回”的必填门禁
- “驳回原因”字段补充了退回第 11 步并生成新轮次的说明
- “评审意见”字段补充了整改要求与整改责任人录入口径提示
- 历史记录下方新增时间线卡片，便于回看驳回与退回轨迹

### 第 17 步月度评审台账

- 月份实例卡按状态着色：待处理、进行中、已完成、逾期
- 周期详情新增摘要卡，突出本月状态、计划日期、完成日期、关联评审数
- 评审绑定规则以 `FeedbackBanner` 明示，避免“为什么看不到别月记录”的歧义
- 台账表格改为固定表头 + 滚动壳

### 第 18 步颜色退出页

- 顶部新增年产量、阈值、系统建议、人工结论四格摘要
- 退出表单增加“保存记录”和“完成颜色退出”的职责说明
- 人工结论新增退出/保留/延期观察提示卡
- “完成颜色退出”改为危险按钮，和保存记录形成明确区分

### 流程页 / 看板 / 日历

- 动作按钮按 approve/complete/reject 做视觉分层
- 截止日历明确说明只展示 workflow 节点截止时间
- recurring task 月度实例继续统一收口在评审页台账，不在日历重复投影

## 仍保留的边界

- MVP 当前没有单独的“整改责任人”字段，R13 只在表单提示中明确要求把责任人写入评审意见；不新增字段以避免突破冻结规则。
- recurring task 没有并入流程截止日历，而是通过统一说明保持展示边界一致。

## 浏览器回归入口

- 本地一键执行：`pnpm playwright:test`
- 原生 Playwright 执行：`pnpm --filter @feishu-timeline/web playwright:test:raw`
- 文档和脚本默认会先启动本地 `postgres` / `redis`、执行 `migrate deploy` 与 `seed`
